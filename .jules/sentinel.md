## 2026-06-23 - Path Traversal via Compromised Network Device Hostname
**Vulnerability:** A path traversal vulnerability existed in `juniper_crawler.py` where the hostname retrieved from the scanned switch (`device_data["hostname"]`) was directly used to construct the filename for the switch's configuration backup (`f"{filename_hostname}_backup_{timestamp}.cfg"`).
**Learning:** If a network device is compromised or maliciously configured with a hostname like `../../../etc/pwned`, the backup file could be written outside the intended `outputs/` directory, potentially overwriting critical system files or allowing for arbitrary file creation on the crawler's host system. Never trust data returned from external devices, even administrative ones.
**Prevention:** Always sanitize variables that are incorporated into file paths, especially those derived from external network responses. Replace all characters except alphanumeric, underscores, dots, and hyphens with a safe character (like `_`) before using them in `os.path.join`.
